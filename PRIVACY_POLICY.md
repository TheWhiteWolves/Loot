# Privacy Policy

## Loot Discord Bot

**Last Updated:** April 17, 2026

### 1. Introduction

This Privacy Policy ("Policy") describes how the Loot Discord Bot ("Bot") collects, uses, and protects data related to its users and Discord server members. Your privacy is important to us, and we are committed to being transparent about how we handle information.

### 2. Information We Collect

#### 2.1 User IDs and Discord Account Information

When you use the Bot, we collect and temporarily store:

- **Discord User ID** - Used to identify participants in loot sessions
- **Discord Username** - Displayed in loot results and winner announcements
- **Discord Avatar/Profile Data** - May be used for UI display purposes

#### 2.2 Loot Session Data

For active loot sessions, we store:

- **Item Name** - The name of the item being rolled for
- **Participant List** - User IDs of all participants in the session
- **Random Rolls** - Numbers generated for each participant (1-100)
- **Winner Information** - User ID and roll result of the winner
- **Message ID** - Discord message identifier for the loot roll

#### 2.3 Command Usage Data

We may collect:

- **Command Names** - Which Bot commands are used (`/loot`, `/ping`, `/sync`)
- **Command Frequency** - How often commands are used
- **Timestamp Information** - When commands are executed (for logging purposes)

### 3. How We Use Information

We use collected information for:

- **Primary Purpose:** Operating the Bot and providing loot rolling functionality
- **Service Improvement:** Identifying bugs and improving Bot performance
- **Logging:** Recording errors and issues for debugging purposes
- **User Experience:** Displaying accurate results to users

**We do not use your information for:**

- Commercial purposes or advertising
- Selling or sharing data with third parties
- Building marketing profiles
- Tracking behavior outside the Bot's functionality

### 4. Data Storage

#### 4.1 Temporary Storage

Loot session data is stored **only in the Bot's runtime memory** while the Bot is active:

- Data persists only during active loot sessions
- Data is **automatically cleared** when the Bot restarts
- Data is **deleted** when a loot session ends

#### 4.2 No Persistent Database

The Loot Bot does **not**:

- Use external databases for data storage
- Store historical loot session data
- Keep permanent records of rolls or results
- Archive user participation histories

#### 4.3 Discord's Servers

All communication goes through Discord's servers. See [Discord's Privacy Policy](https://discord.com/privacy) for information on how Discord handles data.

### 5. Data Sharing

Your data is **not shared** with:

- Third-party services or APIs
- Marketing companies
- Data brokers
- Advertisers
- Any external organizations

**Data may be accessed by:**

- The Bot developers (for debugging and maintenance)
- Your Discord server administrators (who can see Bot messages in channels)

### 6. User Control

You have control over your data:

- **Do Not Participate:** You can choose not to use the Bot or join loot sessions
- **Leave Servers:** Removing the Bot from your server stops data collection
- **Clear Data:** Bot restart automatically clears all session data

**You cannot:**

- Request historical data recovery (it doesn't exist)
- Export loot session records (they're not stored)
- Opt-out of command logging while using the Bot

### 7. Children's Privacy

The Loot Bot is not intended for children under 13 years old. We do not knowingly collect personal information from children. Discord itself is not available to users under 13, so this Bot is not accessible to that age group.

### 8. Security

We implement security measures to protect your data:

- **Minimal Data Collection** - We collect only necessary information
- **Temporary Storage** - Data is not persistently stored
- **Access Controls** - Only developers can access runtime data
- **Discord Integration** - All communication is encrypted through Discord's API

**However, we cannot guarantee absolute security** due to the nature of internet communication and open-source software.

### 9. Open Source Code

The Loot Bot is open-source software. This means:

- **Transparency** - You can review the code to see exactly what data is collected
- **Community Audit** - The code can be reviewed by security researchers
- **No Hidden Data Collection** - You can verify that we're not collecting data beyond what's disclosed

We encourage users to review the [GitHub repository](https://github.com) for full transparency.

### 10. Discord's Privacy

The Bot operates as part of Discord's ecosystem. Please be aware:

- Discord collects data as described in [Discord's Privacy Policy](https://discord.com/privacy)
- Messages and data are subject to Discord's policies
- Discord may retain messages and metadata according to their policies
- The Bot's operation depends on Discord's infrastructure

### 11. Changes to This Policy

We may update this Privacy Policy at any time. Changes will be posted to this document with an updated "Last Updated" date. Your continued use of the Bot constitutes acceptance of updated policies.

### 12. Compliance and Standards

The Loot Bot complies with:

- **Discord's Developer Terms of Service**
- **Discord's Developer Policy**
- **General data protection principles**

We follow best practices for open-source software and prioritize user privacy.

### 13. Data Deletion

To request deletion of any personal data in the Bot's memory:

1. **Stop Using the Bot** - Remove it from your server or don't participate in loot sessions
2. **Bot Restart** - All session data is cleared when the Bot restarts
3. **Contact Developers** - For specific concerns, reach out through GitHub or community channels

**Note:** Since data is not persistently stored, deletion requests are unnecessary after the Bot restarts or a session ends.

### 14. International Considerations

If you are accessing the Bot from outside the United States:

- Understand that your data may be processed in different jurisdictions
- Review Discord's Privacy Policy for their data handling practices
- Be aware of your local data protection regulations

### 15. Contact and Questions

If you have questions or concerns about this Privacy Policy:

- **GitHub Issues:** Report concerns on the project's GitHub
- **Discord Community:** Ask in community channels
- **Contact Developers:** Reach out through available channels in the repository

### 16. Disclaimer

**This is a hobby/open-source project.** Data privacy measures are implemented to the best of our ability, but we do not guarantee:

- Military-grade security
- Compliance with all international data laws
- Enterprise-level data protection
- Legal representation for data disputes

Users are responsible for understanding Discord's policies and terms when using third-party Bots.

### 17. Effective Date

This Privacy Policy is effective as of April 17, 2026, and supersedes any previous privacy statements.
